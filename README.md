# فاست بلاك تايجر لقطع الغيار — Fast Black Tiger Parts

مرحبًا! هذا الريبو يحتوي على scaffold أولي لتطبيق جوال متعدد المنصات مبني بـ Flutter ومهيأ للعمل على iOS و Android، مع إعدادات لربط Firebase لاحقًا.

الخيارات التي اخترتُها نيابةً عنك:
- اسم التطبيق (عربي): فاست بلاك تايجر لقطع الغيار
- اسم التطبيق (إنجليزي): Fast Black Tiger Parts
- تقنية التطوير: Flutter (واحد للتثبيت على iOS وAndroid)
- باك‑إند وبيانات: Firebase (Firestore, Auth, Storage)
- لغة الواجهة: العربية افتراضيًا (RTL)
- تسجيل الدخول: رقم الهاتف (OTP) كافتراضي
- الدفع: تبدأ بدون بوابة دفع (دفع لاحقًا). يمكن إضافة بوابة لاحقًا.
- لوحة إدارة: واجهة إدارة مبسطة داخل التطبيق (يمكن توسيعها لاحقًا أو بناء لوحة ويب)

ما يوجد في هذا commit الأولي:
- README بالعربية مع تعليمات الربط بـ Firebase وتشغيل المشروع.
- ملف pubspec.yaml أساسي يُحدد تبعيات Flutter و Firebase.
- ملف lib/main.dart مع scaffold للتطبيق (دعم RTL، شاشات: الرئيسية، قائمة القطع، تفاصيل قطعة، سلة، تسجيل دخول).
- .gitignore مهيأ لمشروعات Flutter.

خطوات لاحقة أقترح تنفيذها بعد استلامك لهذا scaffold:
1. تهيئة مشروع Firebase (إنشاء مشروع، تفعيل Authentication (Phone), Firestore, Storage) وحفظ ملفات التهيئة (GoogleService-Info.plist و google-services.json).
2. إضافة مفاتيح التهيئة إلى android/ و ios/ وفقا لتعليمات README.
3. تنفيذ شاشات الإدارة لإضافة/تعديل قطع الغيار أو إنشاء لوحة ويب بسيطة (مثلاً باستخدام Next.js أو لوحة Firebase Console).
4. لاحقًا: ربط بوابة دفع، إضافة اختبارات، ونشر على Play Store و App Store.

لتشغيل المشروع (بعد ربط Firebase):
1. تثبيت Flutter: https://flutter.dev
2. تشغيل: flutter pub get
3. تشغيل على محاكي أو جهاز: flutter run
